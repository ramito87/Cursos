Git: Base de Conocimiento

Guía visual y práctica para dominar Git. Comandos, conceptos y ejemplos listos para copiar.

¿Qué es Git y para qué sirve?

Sistema de control de versiones distribuido. Permite gestionar cambios en archivos y colaborar. 

# Git rastrea cambios en archivos
# Ideal para proyectos de desarrollo, documentación y colaboración

Instalación en Linux y Windows

Instala Git según tu sistema operativo.

# Debian/Ubuntu
sudo apt install git

# Windows
Descargar desde https://git-scm.com

Configurar usuario

Establece tu identidad para los commits.

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
git config --list

Crear y clonar repositorio

Inicia un proyecto o clona uno existente.

# Crear repositorio
git init

# Clonar repositorio
git clone git@github.com:usuario/repositorio.git

Espacio de trabajo

Verifica estado y diferencias.

git status
git add archivo.txt
git diff

.gitignore

Ignora archivos que no deben rastrearse.

# Ejemplo de .gitignore
*.log
node_modules/
.env

Commits y versiones

Guarda cambios y revisa historial.

git commit -m "mensaje"
git log

Reset y checkout

Revertir o cambiar de rama.

git reset --soft HEAD^
git checkout rama

Deshacer y volver a commits

Revertir cambios o volver a estados anteriores.

git revert 
git checkout

Buscar commits

Filtra por mensaje o autor.

git log --grep="mensaje"

Ramas

Trabaja en paralelo sin afectar la rama principal.

git branch
git checkout -b nueva-rama
git merge rama

Stash

Guarda cambios temporales.

git stash
git stash list
git stash clear

Cherry-pick

Aplica un commit específico a otra rama.

git cherry-pick

Resolver conflictos

Detecta y resuelve conflictos de fusión.

# Edita archivos marcados
# Luego:
git add archivo
git commit

Repositorios remotos y GitHub

Sincroniza con servidores externos.

git remote -v
git push origin master
git pull origin master
